BD3-LM Reference Code Summary & Implementation Comparison

2026-07-16 · Reference: github.com/kuleshov-group/bd3lms (Arriola et al., "Block Diffusion: Interpolating Between Autoregressive and Diffusion Language Models", ICLR 2025) · Local: models_seq/bd_models.py, models_seq/bd_trainer.py, main_bd.py

What "passed the comparison verification" means, precisely. The reference repository was cloned and the five core components of the mask kernel (attention mask, vectorized training, SUBS loss, first-hitting sampler, semi-AR generation loop) were compared side-by-side at the code level; all five compute the same mathematics and algorithms as the original. What this verification does not mean: (1) no numerical-equivalence test was run (loading identical weights into both implementations and diffing outputs is impossible — the two codebases use different vocabularies and conditioning); (2) it does not mean every feature of the original was ported — EMA, clipped schedules, etc. were deliberately dropped and are all listed in §4; (3) the graph (uniform-state) kernel was never meant to be a bd3lms port — it is this repository's own design applying the seq_models.py loss per block (per CLAUDE.md instructions).

1. The problem BD3-LM solves, and the core idea

Background. Autoregressive (AR) models achieve high likelihood and fast KV-cached decoding but must emit tokens strictly one at a time. Discrete diffusion (e.g. MDLM) allows parallel generation but lags in likelihood and is tied to a fixed-length canvas, making variable-length generation awkward. BD3-LM interpolates between the two at the block level.

Block decomposition. The sequence is split into blocks of size B; the likelihood factorizes autoregressively across blocks while each block's interior is modeled by masked discrete diffusion:

log p(x) = Σ_b log p(x^b | x^{<b})          ← across blocks: autoregressive
p(x^b | x^{<b}) : within a block: discrete diffusion (NELBO conditioned on previous blocks)
  • B = 1 recovers pure AR; B = sequence length recovers pure diffusion — block size is the AR↔diffusion interpolation knob.
  • Generation is semi-autoregressive: finish one block by diffusion, commit it, append the next noised block. This structure is what enables variable-length generation and a cross-block KV cache.

2. Key parts of the reference code (by file / function)

2.1 Vectorized training — all blocks in one forward pass

diffusion.py :: _forward_pass_diffusion. A naive implementation would need one forward per block (cost × number of blocks). BD3-LM instead feeds the concatenation [xt ‖ x0] (length 2n) in a single pass and controls information flow with a specialized attention mask. The noise level t is sampled independently per block (_sample_t: one uniform sample per block, then repeat_interleave).

2.2 The specialized attention mask — OR of three sub-masks

models/hf/modeling_bd3lm.py :: block_diff_mask. Each query/key is classified by which half it lives in (x0_flag) and its block index:

M_BD  (block diagonal)      : same half & same block            → self-attention inside a noised block
M_OBC (offset block causal) : x_t query → x_0 keys of strictly earlier blocks → clean context conditioning
M_BC  (block causal)        : x_0 query → x_0 keys of earlier-or-equal blocks → updates clean (cacheable) states
mask = M_BD | M_OBC | M_BC

2.3 Loss — MDLM-style NELBO with SUBS parameterization

diffusion.py :: _subs_parameterization, _loss. The denoiser predicts x̂0 (logits over the vocabulary). SUBS (a) forces the MASK token's probability to zero (logits[:,:,mask_index] = -inf), and (b) carries over already-revealed (unmasked) positions as one-hot (their log-prob becomes 0 → no loss contribution). Under the log-linear schedule the NELBO reduces to a simple weighted CE:

L = E_{t~U[ε,1]} [ -(1/t) · Σ_{masked ℓ} log p_θ(x_0^ℓ | x_t) ]     (t independent per block)

2.4 Sampling — semi-AR loop + first-hitting sampler

diffusion.py :: _semi_ar_sampler, _ddpm_caching_update. The outer loop appends one block at a time; the inner loop denoises the current block. The first-hitting sampler (the equivalent, faster sampler of Zheng et al. 2025) jumps directly to the time at which the next token is revealed:

u ~ U(0,1);  t ← t · u^(1/num_masked)      ← first-hitting time update
p_x0 = softmax(denoiser(x));  pick ONE masked position uniformly, sample it from p_x0

→ one forward pass per revealed token; a block finishes in exactly B reveals. On top of this sit nucleus (top-p) sampling, KV-cache storage once a block completes, and EOS/entropy-based stopping (_check_stop_conds).

2.5 Variance reduction — the paper's key training techniques

  • Clipped noise schedules (config.algo.var_min=True, on by default): t is sampled only from an interval [ε_min, ε_max] ⊂ [0,1]. During training the interval that minimizes the variance of the validation loss is searched periodically (_clipped_schedule_search, delta 0.05). This is a headline contribution of the paper — it substantially reduces gradient variance and improves perplexity, and matters most at small block sizes.
  • Antithetic t sampling: stratifies t across the batch and blocks to reduce estimator variance.
  • EMA (0.9999): an exponential moving average of the weights is used for evaluation and generation.

2.6 Backbone — DDiT (rotary + adaLN)

modeling_bd3lm.py :: DDiTBlock. Pre-LN transformer with rotary position embeddings; the noise level σ is embedded by a TimestepEmbedder and injected via adaLN (zero-init; 6 modulations per block: shift/scale/gate × attn/MLP). For masked diffusion, however, masked tokens themselves reveal the noise level, so time conditioning is effectively unnecessary (stated in the paper), and the conditioning vector c is broadcast once per sequence.

A. Domain adaptation (language modeling → OD route planning)

D. Local-only addition — the graph (uniform-state) kernel

This kernel does not exist in bd3lms; it is a composition, not a port: BD3-LM's block machinery (attention masks, [xt‖x0] training, semi-AR generation) is kept, while the within-block noising and loss are replaced by this repository's uniform-state CTMC diffusion from seq_models.py (Destroyer/Restorer), per block-diffusion-loss.pdf and CLAUDE.md.

  • Noising: Qt = exp((A′−D′)βt) — a CTMC built from the road-graph adjacency; the state space is V+1 with an added <end> state (virtual <end> node of degree bd_eos_deg).
  • Loss: D3PM ELBO = KL[q(xt−1|xt,x0) ‖ pθ] + CE + connectivity regularizer (the same math as Restorer.forward, applied per block under the loss mask).
  • Generation: full max_T (100)-step reverse per block with an MC posterior approximation, followed by an adjacency-constrained decode.
  • Time conditioning forced on (uniform noise is not self-identifying from xt alone).

5. Summary

  • The mask kernel follows BD3-LM's mathematics (mask · loss · sampler) exactly — all five components in §3 verified by side-by-side code comparison.
  • The differences fall into three buckets: (A) task adaptation (OD conditioning · length spec · stop rule — required by the project spec), (B·C) removed training conveniences (EMA · clipped schedules · antithetic — second-order effects on accuracy, none on within-sweep comparisons), and (D) the added graph kernel (an original design outside bd3lms).
  • One caveat for interpreting results: the absence of clipped schedules may make small block sizes look worse than the original recipe would. Trends across block sizes remain valid, but avoid absolute claims like "blk1 equals the paper's AR baseline."
